Reward Function

What the PPO agent optimises — and what MPC scores each trajectory against

Formula

// How far outside the comfort band are we?
deviation = max(0, 18 − T_inside) + max(0, T_inside − 24)

// Comfort term: reward if inside band, linear penalty if outside
r_comfort = +1.0 if deviation == 0
= −deviation otherwise

// Energy term: small penalty for running equipment
r_energy = −(0.05 × fan_on + 0.10 × heater_on)

r(t) = r_comfort + r_energy

Worked examples — what the agent actually experiences

T_inside = 21°C, fan=0, heater=0

deviation = 0

r = +1.0

Comfortable but fan needed

T_inside = 23°C, fan=1, heater=0

deviation = 0

r = 1.0 − 0.05 = +0.95

Slightly too hot, fan on

T_inside = 27°C, fan=1, heater=0

deviation = 3

r = −3.0 − 0.05 = −3.05

Midsummer — no action taken

T_inside = 40°C, fan=0, heater=0

deviation = 16

r = −16.0

January deep freeze, heater off

T_inside = 5°C, fan=0, heater=0

deviation = 13

r = −13.0

January, heater brings it in range

T_inside = 20°C, fan=0, heater=1

deviation = 0

r = 1.0 − 0.10 = +0.90

Why these numbers

  • Linear (not quadratic) comfort penalty — interpretable and stable during RL training. 5°C violation = −5 per step. Easy to reason about.
  • Energy weight 0.05 / 0.10 — energy is a tiebreaker, not a constraint. The agent will never sacrifice comfort just to save energy. A 1°C comfort violation (−1) is 10–20× worse than running the fan (−0.05).
  • Heater costs 2× fan — heater draws more power (up to 17 kW vs 12 kW in the data), so its energy penalty is proportionally higher.
  • Same formula for MPC scoring — MPC sums r(t) across all 24 simulated steps to score each candidate schedule.
