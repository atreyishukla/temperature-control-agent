System Architecture — Option A

Everything in Python · PyTorch + Stable-Baselines3 · Flask HTTP server for Node-RED

Three training phases (must happen in order)

1

LSTM World Model

What it does: Learns the physics of the building from historical data. Given the last 24 hours of [T_outside, T_inside, T_floor, solar, fan, heater], predicts T_inside and T_floor one hour ahead.

Training: Supervised learning on Concrete_floor_results.xlsx · MSE loss · Adam optimizer · ~30 epochs

Architecture: 2-layer LSTM (hidden=128) → Linear → 2 outputs (T_inside_next, T_floor_next)

2

PPO Agent

What it does: Learns a reactive policy — given current state, output fan on/off and heater on/off. Trains entirely inside a simulated Gym environment that uses the frozen LSTM as its physics engine.

Training: Reinforcement learning · Stable-Baselines3 PPO · ~500k timesteps · reward = comfort + energy penalty

Actions: Discrete(4) — {off/off, fan/off, off/heater, fan/heater}

3

MPC Optimizer

What it does: Every hour, samples 1024 random 24-step fan/heater schedules, simulates each one using the LSTM, scores each by reward, picks the best first action. Thinks 24h ahead.

Method: Random shooting (correct for discrete on/off actions) · runs in ~100ms on CPU

No separate training needed — uses the frozen LSTM at inference time

Runtime loop (every hour)

● Real sensors → last 24h window (144 values)
↓
● MPC: sample 1024 random 24h schedules → simulate each with LSTM → pick best → mpc_action
↓
● PPO: forward pass on current state → ppo_action
↓
● Blend: 70% MPC + 30% PPO → final discrete fan/heater command
↓
● Send to actuators · log experience · (every 24h: re-train PPO on new data)

File structure

lstm_ppo_agent/
data/ ← Concrete_floor_results.xlsx
models/ ← saved LSTM + PPO checkpoints (.pt files)
train_lstm.py ← Phase 1: supervised training
train_ppo.py ← Phase 2: RL training in LSTM sim
mpc.py ← random shooting MPC (used at runtime)
server.py ← Flask HTTP server (Node-RED talks to this)
experience.csv ← real sensor log for continual learning
