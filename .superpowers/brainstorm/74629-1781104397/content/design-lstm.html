LSTM World Model — Architecture & Data Pipeline

Learns building physics from 8760 hours of Edmonton data · Predicts one step ahead · Used by both MPC and PPO env

Data pipeline (before any training)

Step 1 Load Excel Results sheet → 8760 rows, 7 columns
Step 2 Binarise actions: fan_on = (Cooling_power > 0), heater_on = (Heating_power > 0)
→ 6 features: T_outside, T_inside, T_floor, SR_direct, fan_on, heater_on
Step 3 Time-based split (NO shuffling — time series)
Train: hours 0–6131 (70% = ~8.5 months)
Val: hours 6132–7315 (15%)
Test: hours 7316–8759 (15%)
Step 4 Fit StandardScaler on training set only → save as scaler.pkl (no data leakage)
Scales T_outside, T_inside, T_floor, SR_direct to zero mean / unit std
fan_on and heater_on stay as 0/1 (already bounded)
Step 5 Sliding window → sequences of length 24
X[t]: features[t-24 : t] shape (24, 6)
y[t]: [T_inside[t], T_floor[t]] shape (2,) — also normalised
Result: 6108 train sequences, 1294 val, 1444 test

How MPC uses it (autoregressive rollout)

# MPC needs to simulate 24 steps ahead for each candidate schedule
window = last_24h_real_data.copy() # shape (24, 6)
for step in range(24):
T_next = lstm.predict(window) # → [T_inside, T_floor]
new_row = [T_outside_forecast[step], # from weather API
T_next[0], T_next[1], # predicted temps
SR_forecast[step], # from weather API
fan_schedule[step], # candidate action
heater_schedule[step]]
window = slide(window, new_row) # drop oldest row, append new

⚠ Errors compound over 24 steps — this is expected and normal. The LSTM only needs to be good enough that the MPC can distinguish good schedules from bad ones.
