Reward Function — Hard Version

Prevents "always off" · Asymmetric cold/hot · Inaction explicitly punished

Formula

# How far outside the comfort band
cold_dev = max(0, 18 − T_inside) # °C below 18
hot_dev = max(0, T_inside − 24) # °C above 24

# 1. COMFORT — quadratic, asymmetric (cold hurts 3× more than hot)
r_comfort = +2.0 if in [18,24]
= −(cold_dev²) × 3.0 if cold
= −(hot_dev²) × 1.0 if hot

# 2. INACTION — fires when deviation > 3°C AND the right device is off
# This is what stops the agent from doing nothing and hoping for the best
r_inaction = −10 × cold_dev if cold_dev > 3 AND heater_on == 0
= −4 × hot_dev if hot_dev > 3 AND fan_on == 0
= 0 otherwise

# 3. ENERGY — tiny tiebreaker, never overrides comfort
r_energy = −(0.05 × fan_on + 0.10 × heater_on)

r(t) = r_comfort + r_inaction + r_energy

Worked examples

Why this works

  • Quadratic comfort — deviations compound. 2°C = −12/step. 5°C = −75/step. Agent cannot ignore this.
  • Cold 3× hot — in Edmonton, a building at −8°C (seen in the data) is a much harder emergency than 40°C. Pipes, safety.
  • Inaction penalty threshold at 3°C — small 1–2°C excursions don't trigger it (LSTM uncertainty margin), but clear violations get the extra kick.
  • Inaction only penalises the relevant device — cold + fan on still gets inaction penalty (fan doesn't heat). Agent learns which device solves which problem.
  • Energy cost is ~0.5% of a 3°C comfort penalty — agent will never sacrifice comfort to save energy.
